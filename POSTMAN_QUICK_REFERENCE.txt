═══════════════════════════════════════════════════════════════════════════════

Step 1: Import Collection
┌──────────────────────────────────────────────────────────────────────────┐
│ 1. Open Postman                                                          │
│ 2. Click "Import" button (top left)                                      │
│ 3. Select PostmanCollection.json file                                    │
│ 4. Click "Import"                                                        │
└──────────────────────────────────────────────────────────────────────────┘

Step 2: Login & Get Token
┌──────────────────────────────────────────────────────────────────────────┐
│ 1. Expand "Authentication" → "Login"                                     │
│ 2. Click "Send"                                                          │
│ 3. Copy the "token" value from response                                  │
│ 4. Go to Collections → Variables                                         │
│ 5. Paste token in "token" field value                                    │
│ 6. Click "Save"                                                          │
└──────────────────────────────────────────────────────────────────────────┘

Step 3: Test Doctor Endpoints
┌──────────────────────────────────────────────────────────────────────────┐
│ 1. Expand "Doctors" section                                              │

│ 3. Review the request (modify if needed)                                 │
│ 4. Click "Send"                                                          │
│ 5. Check response status & body                                          │
└──────────────────────────────────────────────────────────────────────────┘

🔧 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: 401 Unauthorized
→ Solution: Login first and update {{token}} variable

Issue: 500 Internal Server Error
→ Solution: Ensure Doctor table exists in database (see QUICK_FIX.txt)

Issue: Base URL not working
→ Solution: Update {{baseUrl}} variable to match your server
   Default: https://localhost:5001/api
   Change if using different port or server

Issue: Cannot find endpoint
→ Solution: Make sure you're using the Doctors section
   (Some older collections may not have it)
