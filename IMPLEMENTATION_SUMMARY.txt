🔌 API ENDPOINTS:

   GET ENDPOINTS (Public - All Authenticated):
   ├── GET /api/doctors                    → Get all doctors
   ├── GET /api/doctors/{id}               → Get doctor by ID
   ├── GET /api/doctors/user/{userId}      → Get doctor by User ID
   ├── GET /api/doctors/license/{number}   → Get doctor by License
   ├── GET /api/doctors/specialization/{spec} → Filter by specialty
   └── GET /api/doctors/available          → Get available doctors

   MUTATION ENDPOINTS:
   ├── POST /api/doctors                   → Create (Admin only)
   ├── PUT /api/doctors/{id}               → Update (Admin/Doctor)
   └── DELETE /api/doctors/{id}            → Delete (Admin only)

═══════════════════════════════════════════════════════════════════════════════

✅ VALIDATION IMPLEMENTED:

   Creation:
   ├── UserId must exist and be a Doctor role
   ├── Specialization: 1-100 characters (required)
   ├── LicenseNumber: format A-Z0-9-, max 50 (unique, required)
   ├── YearsOfExperience: 0-70 (required)
   ├── ConsultationFee must be > 0 (required)
   └── Optional fields have max length checks

   Duplicate Prevention:
   ├── One Doctor profile per User
   └── Unique license number constraint
